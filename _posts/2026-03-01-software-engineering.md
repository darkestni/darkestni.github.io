## Lecture 1: 软件工程简介

### 什么是软件工程？

软件工程是将工程学原理应用于软件开发，目的是生产高质量、可靠、可维护的软件。

### 软件危机

- **定义**：软件开发过程中出现的严重问题（延期、超预算、质量差）
- **原因**：需求不明确、缺乏规范流程、项目管理不当
- **解决方案**：引入工程化方法

### 软件质量属性

| 属性 | 说明 |
|------|------|
| 功能性 | 满足用户需求 |
| 可靠性 | 稳定运行，容错能力 |
| 易用性 | 用户界面友好 |
| 效率 | 资源利用率高 |
| 可维护性 | 易于修改和扩展 |
| 可移植性 | 跨平台运行能力 |

## Lecture 2: 软件过程模型

### 常见过程模型

#### 1. 瀑布模型 (Waterfall)

```
需求分析 → 设计 → 编码 → 测试 → 维护
```

- **优点**：结构清晰，文档完整
- **缺点**：不灵活，难以应对需求变化

#### 2. 敏捷开发 (Agile)

- **核心价值观**：个体和互动、可工作的软件、客户合作、响应变化
- **常见方法**：Scrum、XP、Kanban

#### 3. 迭代模型

- 分周期开发，每个周期交付可工作的软件
- 逐步完善功能

### 过程对比

| 模型 | 适用场景 | 优点 | 缺点 |
|------|----------|------|------|
| 瀑布 | 需求明确的项目 | 文档完整 | 不灵活 |
| 敏捷 | 需求变化频繁 | 快速响应 | 文档较少 |
| 迭代 | 大型复杂项目 | 风险可控 | 周期较长 |

## Lecture 3: 需求工程

### 需求类型

1. **功能需求**：系统应该做什么
2. **非功能需求**：系统应该做得怎么样（性能、安全性等）
3. **领域需求**：特定业务领域的约束

### 需求获取方法

- 用户访谈
- 问卷调查
- 观察法
- 原型法

### 需求文档

- **SRS (Software Requirements Specification)**：软件需求规格说明书
- 包含：功能描述、接口定义、性能要求、约束条件

## Lecture 4: 版本控制

### Git 基础

```bash
# 初始化仓库
git init

# 添加文件到暂存区
git add .

# 提交更改
git commit -m "commit message"

# 推送到远程
git push origin main

# 拉取更新
git pull origin main
```

### 分支管理

```bash
# 创建分支
git branch feature-name

# 切换分支
git checkout feature-name

# 合并分支
git merge feature-name
```

### Git 工作流

```
工作区 → 暂存区 → 本地仓库 → 远程仓库
```
